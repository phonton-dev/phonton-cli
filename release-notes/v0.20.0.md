# v0.20.0 — Merge-gate positioning and honest execution labeling

## Summary

Positions Phonton as a proof-carrying merge gate: receipt-first TUI, workspace-aware
Ask, and explicit `execution:` labels for provider vs `local-template` runs.

## Highlights

- Workspace-aware Ask via `ask_context` (repo paths + current goal in prompt).
- Receipt block shown before in-flight worker noise when a handoff exists.
- TUI `execution:` line distinguishes provider, local-template, and mixed runs.
- Docs: `CURRENT.md`, rewritten vision/positioning, getting-started + concepts.
- DeepSeek provider-only benchmark report (`benchmarks/reports/deepseek-2026-06-01-...`).

## Claims

Do not publish token-leaderboard copy without `token_claim_eligible` artifacts.
Local-template runs are reliability evidence, not provider efficiency claims.
