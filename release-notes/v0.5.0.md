# Phonton CLI v0.5.0

Architecture release for prompt handling, command execution, token visibility, and stronger quality gates.

## Added

- Long or multiline pastes collapse into prompt artifacts such as `[paste: 18 lines, 3.4k chars]`, while the bounded full paste stays attached to the submitted goal.
- `Ctrl+V` imports text from the Windows clipboard, including clipboard-history selections made with `Win+V` when the terminal does not emit bracketed paste directly.
- `/run <cmd>` and `!<cmd>` run commands through the sandbox path and show recent command status/output in the TUI.
- Prompt-section token manifests now appear in the Flight Log so prompt waste is easier to inspect.
- Stack-aware preflight adds concrete npm, Cargo, or Makefile verification/run plans when those files exist.

## Changed

- Worker provider calls no longer include the worker system prompt twice.
- Generic `completed: ...` memory records are filtered out of future memory preambles.
- Broad `make chess`-style goals now carry playable-game criteria and fail quality gates before review if the result is only a placeholder.
- Prompt editing gained `Ctrl+U`, `Ctrl+K`, history navigation, and slash-command completion.
