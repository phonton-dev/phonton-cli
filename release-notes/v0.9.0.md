# Phonton CLI v0.9.0

This release focuses on lower-noise prompts, better history, and durable
workspace trust.

## Added

- Structured workspace trust records with permission mode, source, trusted-at,
  and last-seen metadata.
- `/trust current`, `/trust list`, and `/trust revoke-current` in the TUI.
- Prompt history in saved session snapshots so `phonton -r` keeps recent prompt
  recall.
- Filterable TUI History view with selected-row details.

## Changed

- Worker first attempts now use a shorter diff-only system prompt. The bulky
  diff example is included only when retry errors suggest format repair is
  needed.
- Planner/semantic repo context slices are deduplicated before rendering into
  worker prompts.
- Prompt manifests now show repo-code tokens, budget limit, auto-compacted
  tokens, and deduped tokens.

## Validation

Run from a clean public CLI clone:

```text
cargo fmt --all -- --check
cargo test --locked --workspace
cargo clippy --locked --workspace --all-targets -- -D warnings
cargo build --release --locked -p phonton-cli
target/release/phonton.exe version
target/release/phonton.exe --help
npm run test:npm-wrapper
npm pack --dry-run
```
