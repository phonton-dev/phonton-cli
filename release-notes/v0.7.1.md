# Phonton CLI v0.7.1

Clarification hotfix for the v0.7.0 trust-loop release.

## Fixed

- Stackless broad goals such as `make chess` now become `NeedsClarification` before worker dispatch when the GoalContract has unanswered questions and no concrete run or verification plan.
- The TUI now submits goals without awaiting setup work in the key handler, so the prompt stays responsive while planning, memory lookup, extension loading, and local context setup continue.
- The active pane now renders clarification questions directly instead of showing a failed worker run for goals Phonton should not have attempted yet.

## Validation

Local validation available in this Windows shell:

```text
cargo fmt --all -- --check
cargo metadata --locked --no-deps --format-version 1
npm pack --dry-run
```

Blocked locally and expected to run in GitHub Actions:

```text
cargo test --locked --workspace
cargo clippy --locked --workspace --all-targets -- -D warnings
cargo build --release --locked -p phonton-cli
npm run test:npm-wrapper
```

Rust compile/test validation is blocked because this local Windows shell does not have the MSVC linker installed (`link.exe`). The npm wrapper test is blocked until a local Rust binary exists under `target/release` or `target/debug`.
