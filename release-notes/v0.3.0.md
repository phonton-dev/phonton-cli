# Phonton CLI v0.3.0

Phonton CLI 0.3.0 is the Extension Runtime Alpha. It adds local extension
loading, MCP visibility and approval flows, persistent memory commands, stronger
provider diagnostics, and a cleaner release path for measured benchmark claims.

## Highlights

- Added local extension manifests for skills, steering, MCP servers, and
  profiles.
- Added `phonton extensions` inventory and doctor commands, plus `phonton skills
  list` and `phonton steering list` aliases.
- Added `phonton mcp list`, `phonton mcp tools`, and `phonton mcp call`.
- Added lazy MCP server startup, tool discovery, trust checks, approval policies,
  capped tool results, and event reporting.
- Added a TUI MCP approval modal with approve, deny, keyboard navigation, and
  automatic denial on quit.
- Added worker-facing `MCP_TOOL_CALL` handling with an end-to-end approval plus
  verified-diff test.
- Added local decision memory commands for listing, editing, pinning, unpinning,
  and deleting memory records.
- Added Cloudflare Workers AI as a named BYOK provider using the
  OpenAI-compatible endpoint, defaulting to `@cf/moonshotai/kimi-k2.6`.
- Added an explicit Cloudflare account ID field to Settings and config via
  `provider.account_id`.
- Updated the TUI splash logo and gradient treatment.

## Provider Setup

Cloudflare Workers AI can be configured with:

```toml
[provider]
name = "cloudflare"
model = "@cf/moonshotai/kimi-k2.6"
account_id = "<cloudflare-account-id>"
```

Use `provider.api_key` in local config or one of these environment variables:

```text
CLOUDFLARE_API_TOKEN
CLOUDFLARE_WORKERS_AI_API_TOKEN
CLOUDFLARE_API_KEY
```

`provider.base_url` remains available for full endpoint overrides and custom
OpenAI-compatible providers.

## Known Limits

- Extension installation is not a marketplace yet; this release focuses on
  local manifest loading, diagnostics, and execution.
- MCP server coverage depends on user/workspace configuration and trust policy.
- Competitive claims against Codex, Claude Code, Cursor, or other agents require
  the fixed-fixture benchmark protocol and provider-reported token usage.
