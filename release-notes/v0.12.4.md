# Phonton CLI v0.12.4

v0.12.4 is an autonomy-loop hotfix focused on actual token savings during
failed generated-app and repair paths.

## Changed

- Worker retries stop after two matching verifier/parser diagnostic signatures.
- Orchestrator retry diagnostics are included in the next worker prompt and
  semantic-context query.
- Stale hunk failures add repair guidance that tells the worker not to repeat
  the same patch path.
- The Flight Log now shows a compact `repair` event before redispatch, so users
  can see when Phonton is using verifier diagnostics instead of blind retry.
- Long verifier diagnostics are compacted in the Flight Log while remaining
  intact in the raw event record and Problems view.

## Fixed

- Small generated artifact full-file replacement hunks are written directly
  when safe, avoiding fragile patch application against stale removed lines.
- Worker failure text now reports an honest stop before blind retry instead of
  implying a model-tier escalation that did not happen.
