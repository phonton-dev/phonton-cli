# Phonton CLI v0.11.0 - Context Engine

v0.11.0 focuses on Phonton's main competitive metric:

```text
verified success per 10k provider-reported tokens
```

## Added

- Typed `ContextPlan` data for provider calls.
- A deterministic context compiler in `phonton-context`.
- Compact repo-map orientation before selected code slices.
- Prompt manifest buckets for context target, repo-map tokens, selected code
  tokens, and omitted candidate code tokens.
- Attempt-level prompt buckets for first attempt, repair attempts,
  context/artifact tokens, and verifier retry tokens.
- `/ask <question>` for direct stateless questions from the prompt bar.
- `scripts/score-benchmark-runs.ps1` for scoring benchmark run folders by
  verified success per 10k tokens.

## Changed

- Worker prompts are now explicitly token-minimizing and diff-minimal.
- Providers use dynamic completion ceilings instead of large fixed output
  limits, and broad generated-code repair attempts keep enough output headroom
  to avoid repair starvation.
- `/context`, `/why-tokens`, and Flight Log output now show what the context
  compiler included and omitted, including honest target-exceeded status.
- Ask answers are scrollable and render simple headings, bullets, numbered
  lists, code fences, and status lines.
- Receipt and Markdown review output include a short deterministic summary
  generated from the handoff packet without another model call.

## Fixed

- Benchmark scoring no longer double-counts token aliases such as
  `input_tokens`/`prompt_tokens` or `output_tokens`/`completion_tokens`.

## Notes

This release does not make public superiority claims. It adds the architecture
and benchmark metric needed to prove or disprove Phonton's token-efficiency
claims with reproducible artifacts.
