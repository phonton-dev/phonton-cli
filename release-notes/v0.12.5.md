# Phonton CLI v0.12.5

v0.12.5 is a provider reliability release. It fixes the failure mode where a
configured model could pass a generic connection check, return empty or
non-diff output during a goal, and waste repair tokens before failing.

## Highlights

- Provider readiness now uses a tiny parseable unified-diff canary.
- Empty OpenAI-compatible model content is rejected immediately.
- Goal dispatch blocks bad hosted provider/model routes before worker tokens
  are spent, unless `provider.allow_unverified_model = true`.
- `phonton providers list|sync|doctor|import-opencode` exposes the Models.dev
  catalog and provider route checks.
- OpenCode and OpenCode Go are first-class OpenAI-compatible routes via
  `OPENCODE_API_KEY` or `provider.api_key_source = "opencode"`.

## Validation

- `cargo test --locked -p phonton-providers -p phonton-cli --lib`
- `cargo test --locked -p phonton-cli --bin phonton`
