# Phonton CLI v0.6.2

Sandbox and prompt hotfix.

## Fixed

- Fixed worker filesystem tool routing so approval-required sandbox decisions are surfaced instead of silently continuing with direct read/write execution.
- Tightened sandbox path containment by normalizing parent traversal before root and blocked-path evaluation.
- Prevented deleted or cleared prompt paste artifact chips from submitting hidden pasted content later.
- Tightened `/run` command parsing so `/runfoo` is not treated as `/run foo`.
- Routed single-ampersand shell command text through the approval-gated bash path instead of the structured safe-run path.

## Validation

Validated before release:

```text
cargo fmt --all -- --check
cargo clippy --locked --workspace --all-targets -- -D warnings
cargo test --locked --workspace
cargo build --release --locked -p phonton-cli
npm pack --dry-run
```

Rust compile/test validation ran in GitHub Actions because the local Windows
release shell does not have the MSVC linker installed.
