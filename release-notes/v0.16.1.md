# Phonton CLI v0.16.1

v0.16.1 makes extensions installable from the terminal instead of only
inspectable after manual file copies.

## Highlights

- `phonton extensions install <source>` installs `.phonton` extension packs from
  GitHub or local paths.
- Built-in open-source MCP catalog entries install with short ids:
  `context7`, `github`, `chrome-devtools`, `playwright`, `firecrawl`,
  `supabase`, `mongodb`, and `figma`.
- `phonton extensions catalog` lists the catalog ids, source repos, licenses,
  trust levels, permissions, and install commands.
- `phonton extensions new <path> [skill|steering|mcp-server|profile]`
  scaffolds small auditable extension packs.
- `phonton extensions validate` aliases `doctor` for a Gemini-style validation
  command.
- `.phonton/mcp.d/*.toml` files are now loaded, so installed MCP catalog entries
  can live in separate files.
- `phonton goal --prompt-file <path>` runs one exact goal without launching the
  TUI, which gives CI and benchmark harnesses a stable path that does not rely
  on clipboard or PTY paste behavior.
- `phonton benchmark export --latest --format json` now labels token
  comparability with `execution_mode`, `token_usage_source`,
  `provider_call_count`, `token_claim_eligible`, `benchmark_warnings`, and
  cache creation tokens.
- Memory lookup now handles common morphology and a small engineering synonym
  set, then ranks with IDF-weighted overlap so high-frequency words do not hide
  specific constraints.
- Cargo verification uses file-backed locks under `target/.phonton-locks`, so
  separate Phonton CLI processes coordinate on the same worktree target guard.
- Windows sandboxing now warns explicitly when nested Job Objects prevent job
  attachment; the direct child still uses `kill_on_drop`, but descendant cleanup
  is reported as degraded.

## Install Examples

```text
phonton extensions install context7
phonton extensions install github --scope user
phonton extensions install https://github.com/phonton-dev/phonton-review-gate-extension
phonton extensions install phonton-dev/phonton-review-gate-extension --ref main --dry-run
phonton goal --prompt-file prompt.md --yes --permission-mode full-access --timeout-seconds 900 --json
phonton benchmark export --latest --format json
```

## Trust Boundary

Installing a catalog MCP entry writes local config only. Phonton still applies
workspace trust, permission checks, and approval before MCP tools can affect a
run.

## Benchmark Claims

Token-efficiency claims should only use verified provider-backed runs where
`token_claim_eligible` is `true`. Local-template, estimated, mixed, unavailable,
and failed runs remain useful product evidence, but they should not be counted
as provider-token wins.

## Validation

```text
cargo fmt --all -- --check
cargo test --locked --workspace
cargo clippy --locked --workspace --all-targets -- -D warnings
cargo build --release --locked -p phonton-cli
target/release/phonton.exe version
target/release/phonton.exe --help
target/release/phonton.exe extensions catalog
target/release/phonton.exe extensions install context7 --dry-run
target/release/phonton.exe goal --prompt-file prompt.md --yes --permission-mode full-access --timeout-seconds 900 --json
npm run test:npm-wrapper
npm pack --dry-run
```
