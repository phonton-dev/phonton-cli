# Phonton CLI v0.16.0

v0.16.0 is the addressable-context release. It makes Phonton's ADE loop more
inspectable before work starts and more honest after work finishes.

## Highlights

- Typed `@...` context mentions now cover files, directories, symbols, MCP
  servers, and MCP tools.
- `/context` and the Context focus surface show resolved, missing, and
  approval-gated mention rows before broad work gets buried in a run.
- `phonton why-tokens --by-source` now separates local prompt estimates from
  provider-reported billing usage and renders local-only runs as
  `no provider call`.
- Prompt manifests now carry resolved `@...` mention rows plus
  attribution-only mention token totals so file mentions stay visible without
  double-counting attachment or MCP/tool prompt buckets.
- `phonton extensions doctor` now warns on networked or mutating MCP trust even
  when a server has not listed explicit permissions.

## Context Mentions

Supported forms:

```text
@README.md
@src/main.rs
@docs/
@"docs/release note.md"
@[docs/release note.md]
@symbol:GoalContract
@mcp:filesystem
@mcp:filesystem/read_file
```

Directory mentions are bounded summaries. MCP mentions resolve configured
capability metadata only; Phonton does not start or call MCP servers just
because a goal mentions them.

## Claim Boundary

Phonton is designed to compete with Cursor, Claude Code, Codex, HermesAgent,
and BridgeSpace on proof-carrying ADE behavior: visible plans, addressable
context, verification gates, typed handoffs, and inspectable memory. This
release does not publish a benchmark superiority claim. Those claims require
fixed fixture repos, pinned prompts, tool versions, raw logs, final diffs,
verification logs, and review artifacts.

## Validation

Release validation should run from a clean public CLI clone:

```text
cargo fmt --all -- --check
cargo test --locked --workspace
cargo clippy --locked --workspace --all-targets -- -D warnings
cargo build --release --locked -p phonton-cli
target/release/phonton.exe version
target/release/phonton.exe --help
npm run test:npm-wrapper
npm pack --dry-run
```
