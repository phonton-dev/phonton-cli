# Phonton CLI v0.10.0

v0.10.0 is a trust upgrade release focused on preventing false review-ready
states and making failures easier to inspect and repair.

## Highlights

- Multi-language static syntax verification now runs before review-ready status
  for Rust, Python, JavaScript, TypeScript, JSON, TOML, YAML, HTML, and CSS.
- Existing files are reconstructed from workspace content plus proposed hunks
  before parsing, so partial edits are checked against their post-diff form.
- Failed goals now default to a Problems focus view with concise diagnostics,
  file/layer context, and repair guidance.
- Added `/problems`, `/diagnostics`, `/retry`, `/repair`, and `/why-tokens`.
- `phonton review latest --markdown` can now export failed/unverified receipts
  with diagnostics instead of only verified payloads.
- Verifier retry prompts now carry compact diagnostic context, reducing retry
  token waste.

## Validation

Release validation is expected to run from a clean public CLI clone:

```text
cargo fmt --all -- --check
cargo test --locked --workspace
cargo clippy --locked --workspace --all-targets -- -D warnings
cargo build --release --locked -p phonton-cli
target/release/phonton.exe version
target/release/phonton.exe --help
npm run test:npm-wrapper
npm pack --dry-run
```

No benchmark superiority claim is included in this release.
