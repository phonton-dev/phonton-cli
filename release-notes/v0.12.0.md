# Phonton CLI v0.12.0

v0.12.0 is the verified-success-per-token release. It strengthens the ADE loop
around measurable outcomes: goal contracts, bounded context, route/risk
visibility, runtime proof hooks, surgical repair, and benchmarkable ledgers.

## Highlights

- New `phonton benchmark export --latest --format json` command exports
  OutcomeLedger evidence with provider-reported token buckets, context buckets,
  verification state, quality gates, and final benchmark status.
- Goal contracts now carry structured intent classification, acceptance slices,
  and token policy.
- Broad generated app/game goals such as static HTML chess are treated as
  high-risk: preflight first, runtime verification hooks, capped first attempt,
  and no broad automatic repair.
- Generated app/game work now dispatches as sequential acceptance-slice
  subtasks instead of one broad prompt. Worker prompt budgets are task-class
  capped, semantic retrieval is smaller for low-context classes, MCP result
  context is capped, and provider output ceilings are lower.
- New top-level `phonton why-tokens --by-source` command exposes selected code,
  omitted candidates, memory, artifacts, retry diagnostics, tool output,
  deduped, and cached token buckets.
- New `phonton proof export --latest --format json` command exports the latest
  proof bundle from the OutcomeLedger.
- New `phonton context eval <fixture>` and
  `phonton context diff --indexed --non-indexed <fixture>` commands make
  context-selection behavior testable before benchmark runs.
- Optional Playwright runtime verification hooks can prove generated web
  artifacts when Playwright is installed; otherwise Phonton reports the result
  as unverified instead of claiming success.

## Validation

Before publishing, run the normal clean-clone validation:

```powershell
cargo fmt --all -- --check
cargo test --locked --workspace
cargo clippy --locked --workspace --all-targets -- -D warnings
cargo build --release --locked -p phonton-cli
target/release/phonton.exe version
target/release/phonton.exe --help
npm run test:npm-wrapper
npm pack --dry-run
```

Benchmark claims still require fixed fixtures, exact prompts, raw logs, final
diffs, verification logs, quality review artifacts, and exported benchmark JSON.
