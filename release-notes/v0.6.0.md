# Phonton CLI v0.6.0

Command UX and trust-loop adoption release.

## Added

- Restored slash commands as a first-class TUI surface with a shared command registry, prompt drawer, command palette, and Tab completion.
- Added stable TUI commands for `/settings`, `/config`, `/status`, `/review`, `/memory`, `/permissions`, `/model`, `/commands`, `/goal`, `/task`, `/ask`, `/clear`, `/delete`, `/quit`, and `/exit`.
- Added `/model set <name>` for quick model preference changes.
- Added `phonton init` for first-run config creation.
- Added `phonton demo trust-loop`, a first-run walkthrough of the GoalContract -> verification -> review receipt -> memory flow.

## Fixed

- `/settings` and `/config` no longer get submitted as agent goals.
- Unknown slash commands now show a suggestion instead of queueing accidental work.
- `/run <cmd>` and `!<cmd>` remain sandboxed command execution shortcuts.

## Product Direction

Phonton v0.6.0 tightens the product around one promise: turn every AI code change into an inspectable evidence trail.

The release keeps command UX tied to the ADE loop: goal, contract, edit, verify, review, and remember.
