# Phonton CLI v0.7.4

v0.7.4 is a focused TUI usability release for multi-goal sessions.

## Added

- Stable goal indexes in the sidebar.
- `Alt+Up` / `Alt+Down` to switch goals even while the prompt contains text.
- `Alt+1` through `Alt+9` to jump directly to visible goals.
- `/goals` and `/switch` to open a searchable goal switcher drawer.
- Active panel focus tabs: Receipt, Code, Commands, and Log.
- Code focus for review-ready goals with diff hunks, plus `f` to cycle focus and `[` / `]` to move between changed files.
- `/focus code|commands|receipt|log`, `/copy`, `/rerun`, `/stats`, and `/compress`.

## Changed

- Review-ready goals now default to Code focus when diff hunks are available.
- Command runs are summarized compactly by default and expanded in Commands focus with status, exit code, duration, and stdout/stderr previews.
- `/copy` copies the current focus view text to the Windows clipboard.
- `/rerun` repeats the latest command through the same sandbox command path.

## Validation

Local Rust validation is still blocked in this Codex Windows environment because
`link.exe` and Windows SDK import libraries are unavailable. Local validation:

- `cargo fmt --all -- --check`
- `cargo metadata --locked --no-deps --format-version 1`

Blocked locally by missing MSVC linker/SDK:

- `cargo check --locked -p phonton-cli`
- `cargo test --locked --workspace`
- `cargo clippy --locked --workspace --all-targets -- -D warnings`
- `cargo build --release --locked -p phonton-cli`

GitHub Actions should run the full release gate before publishing.
