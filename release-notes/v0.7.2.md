# Phonton CLI v0.7.2

Goal dispatch hotfix.

## Fixed

- Goal-mode chess requests now dispatch immediately instead of stopping at a clarification state.
- Empty-workspace chess goals now default to a terminal Python game target with:
  - expected file: `chess.py`
  - verification command: `python -m py_compile chess.py`
  - run command: `python chess.py`
- Short chess goals no longer inherit the generic "What exact behavior or artifact should Phonton produce?" clarification question.
- The v0.7.1 nonblocking TUI submission behavior is preserved.

## Validation

Local validation available in this Windows shell:

```text
cargo fmt --all -- --check
cargo metadata --locked --no-deps --format-version 1
npm pack --dry-run
```

Blocked locally and expected to run in GitHub Actions:

```text
cargo test --locked --workspace
cargo clippy --locked --workspace --all-targets -- -D warnings
cargo build --release --locked -p phonton-cli
npm run test:npm-wrapper
```

Rust compile/test validation is blocked because this local Windows shell does not have the MSVC linker installed (`link.exe`). The npm wrapper test is blocked until a local Rust binary exists under `target/release` or `target/debug`.
