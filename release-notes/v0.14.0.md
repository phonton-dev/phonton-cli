# Phonton CLI v0.14.0

v0.14.0 is a proof-and-repair release. It fixes a concrete generated-app
verification bug, hardens Node test command selection so the verifier no
longer hangs on stock Vite/React/Vitest or Jest scaffolds, and clarifies
the failure receipts that surface when Node verification times out.

## Fixed

- `local generated-artifact seed failed verification: npm test timed out after 180s`
  no longer happens for stock Vite/React projects. Node verification now
  rewrites `"test": "vitest"` to `npm test -- --run` and `"test": "jest"`
  to `npm test -- --watchAll=false`, so neither runner enters watch mode.
- Verification `npm` subprocesses always run with `CI=1`,
  `NPM_CONFIG_YES=true`, `NPM_CONFIG_FUND=false`, `NPM_CONFIG_AUDIT=false`,
  `NPM_CONFIG_LOGLEVEL=error`, `NPM_CONFIG_UPDATE_NOTIFIER=false`,
  `NO_COLOR=1`, `FORCE_COLOR=0`, and `NODE_NO_WARNINGS=1`. Stock scaffolds
  do not hang on interactive prompts.
- `test:ci` and `test:run` scripts in `package.json` are preferred over the
  default `test` script during verification. Projects can opt into a
  deterministic non-interactive verification path without changing their
  developer-facing `test` workflow.

## Changed

- Node verification failure receipts include the exact `npm` command
  Phonton attempted (for example `npm test -- --run failed: ...` or
  `npm run build failed: ...`). Users see what to repair without re-running.
- Node test-step timeouts classify as `test harness timeout — likely
  interactive/watch mode` and include compact repair guidance pointing at
  `test:ci`/`test:run` scripts or non-watch invocations. Non-test steps
  (install, build) keep the plain timeout message; their root cause is
  different.

## Added

- `phonton_verify::select_node_test_command` and
  `phonton_verify::npm_verification_env` are now public surface. Future
  verifiers and downstream tooling can reuse the same deterministic Node
  command and environment shape.
- New unit tests cover Vitest rewriting, Jest rewriting, `test:ci`/`test:run`
  precedence, custom script pass-through, missing/empty scripts, and the
  test-harness-timeout error format. None of them wait for a real
  180s timeout to fire — the format is exercised through a pure helper.

## Known Gaps

These items are part of the broader "proof and repair" arc but are not
shipped in this release:

- Browser/runtime verification for generated web artifacts beyond the
  existing optional Playwright path.
- Missing-criteria-only repair behavior so failed goals retry the
  smallest necessary fix instead of broader rework.
- Richer `HandoffPacket` / `OutcomeLedger` / proof export with
  per-command evidence, repair actions, and explicit known-gap surfaces.

## Claims Boundary

This release makes no token-savings or benchmark-superiority claim.
Comparison claims continue to require reproducible benchmark artifacts
captured under fixed prompts, models, and fixtures.
