# Phonton CLI v0.6.1

Cloudflare provider hotfix.

## Fixed

- Fixed Cloudflare Workers AI response parsing so Phonton accepts both strict OpenAI-compatible chat completions and Cloudflare result envelopes.
- Replaced the misleading `missing choices[0].message.content` fallback with an upstream-aware error path for Cloudflare error envelopes.
- Updated Cloudflare chat completion requests to send `max_completion_tokens` and disable provider-side thinking for worker calls, matching the current Workers AI schema for Kimi K2.6 while keeping worker output diff-focused.

## Validation

Validated locally where this Windows environment can compile without the MSVC linker:

```text
cargo fmt --all -- --check
cargo clippy --locked -p phonton-providers --all-targets -- -D warnings
```

Full release validation runs in GitHub Actions because local test binaries require `link.exe` on this machine.
