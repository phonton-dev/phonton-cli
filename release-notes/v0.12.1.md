# Phonton CLI v0.12.1

v0.12.1 fixes the playable chess benchmark path introduced in v0.12.0.

## Fixed

- Empty-workspace chess prompts that explicitly require Vite, TypeScript, and
  React now produce a Vite/React npm app contract instead of a static HTML
  fallback.
- The chess benchmark contract now includes bounded slices for scaffolding,
  chess.js-backed rules, rules tests, React UI, interactions,
  status/history/reset, and concrete npm run/verify commands.
- Later generated-app slices now receive the current artifact snapshot, which
  prevents stale hunk coordinates such as patching line 14 of a four-line file.
- Benchmark acceptance-slice subtasks now use a compact goal label instead of
  repeating the full pasted benchmark prompt in every worker call.
- Successful worker slices retain compact change summaries in context instead
  of the entire previous diff, lowering prompt carryover tokens.
- Vite/npm generated projects now run `npm install`, `npm test`, and
  `npm run build` from a temporary post-diff workspace before review-ready
  status.
- No-git fallback patching now reconstructs existing files from old-side hunk
  coordinates and context/removal lines.
