# Phonton CLI v0.15.0

v0.15.0 is a release-candidate upgrade for the local-first ADE loop:
goal -> plan -> edit -> verify -> review -> remember.

## Added

- Summary-first TUI focus surfaces for Plan, Receipt, Problems, Code,
  Commands, Context, Tokens, and Log.
- TUI `/plan <goal>` previews a GoalContract without running workers, and
  `/approve` starts the selected preview when the plan is acceptable.
- Deterministic proof summaries for plan, work, verification, failure, token,
  context, and handoff evidence.
- Proof, review JSON, and benchmark exports now include the same typed summary
  bundle used by the TUI/review surfaces.
- OutcomeLedger records now preserve context buckets, selected index slices,
  MCP permission evidence, command-run evidence, and summary data.

## Changed

- Missing generated-web runtime proof is surfaced as a known gap and verifier
  finding when a GoalContract planned browser/runtime checks.
- `phonton run latest` records sandboxed command evidence back into the task
  PermissionLedger.
- Token and context summaries remain deterministic and source-attributed; no
  model-generated release summary is required to explain a run.

## Verification

- `cargo fmt --all -- --check`
- `cargo test --locked --workspace`
- `cargo clippy --locked --workspace --all-targets -- -D warnings`
- `cargo build --release --locked -p phonton-cli`
- `target/release/phonton.exe version`
- `target/release/phonton.exe --help`
- `npm run test:npm-wrapper`
- `npm pack --dry-run`

## Claims

This release does not claim a numeric token-savings percentage, competitor
superiority, or "best ADE" status. Public efficiency claims still require fixed
fixtures, pinned prompts, raw logs, provider-reported tokens, verification logs,
and review/proof artifacts.
