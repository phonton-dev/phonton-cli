# Phonton CLI v0.19.5 - Confidence Gate for Under-Specified Goals

v0.19.5 is a trust-loop release for the local-first Phonton ADE. It makes
under-specified goals fail visibly before worker dispatch instead of letting
placeholder code look review-ready.

## Key Improvements

### Goal Contract Confidence Gate

- Broad or vague stackless goals, such as `"make chess"`, now stop at the
  orchestrator intake when the generated `GoalContract` has confidence below
  `70%` or carries unanswered clarification questions.
- The orchestrator emits `TaskFailed`, sets the goal state to `Failed`, and
  populates handoff gaps so the result is explicitly failed/unverified.
- The TUI defaults to Problems focus for this path, giving users a compact
  clarification/repair path before they spend worker retries on a weak plan.

### Why It Matters

The shipped regression covers an observed placeholder loop that spent 12,456
tokens without producing a trustworthy chess implementation. Treat that as
release evidence for this bugfix, not as a broad public token-efficiency claim.
