# Phonton CLI v0.7.3

v0.7.3 focuses on making live runs easier to control and easier to trust.

## Added

- `/context` shows the latest prompt-section token manifest, including system,
  goal, memory/context, attachment, MCP/tool, retry-error, and total estimated
  prompt tokens.
- `/compact` requests an explicit context-compression pass for the selected
  running goal and resets the local prompt context meter.
- `/stop` cancels the selected planning/running goal through the orchestrator
  control channel.
- `/permissions set <mode>` persists local permission modes:
  `ask`, `read-only`, `workspace-write`, and `full-access`.
- The System panel now shows permission mode and latest context-meter status.

## Fixed

- Submitting a goal now pushes an immediate Planning state before slower setup
  work starts, reducing the appearance of TUI input freeze after Enter.
- Hosted providers fail before dispatch when no API key is available, instead
  of falling through to the stub dispatcher.
