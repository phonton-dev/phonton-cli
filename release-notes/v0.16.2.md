# Phonton CLI v0.16.2

v0.16.2 is the benchmark-honest headless reliability release. It tightens the
noninteractive `phonton goal` path used by benchmark harnesses and keeps token
claims tied to complete provider evidence.

## Added

- `phonton goal --prompt-file <path> --json --yes` now captures bounded
  baseline test evidence before editing when the prompt asks to run tests
  first.
- Goal contracts now extract prompt-mentioned files and API signatures, such as
  `src/receipt.js` and `buildReceipt(run)`, into likely files and acceptance
  criteria.
- Prompt-named test commands such as `npm test`, `cargo test`, `pnpm test`,
  `yarn test`, and `pytest` become concrete verify-plan steps.

## Fixed

- Windows headless baseline capture now runs Node package tools through the
  correct command shims, for example `npm.cmd`.
- Nested benchmark work folders no longer checkpoint a parent git repository.
  If the current folder is not a git root, verified hunks are applied directly
  inside that workspace and the receipt reports that no rollback checkpoint was
  recorded.
- Mixed local-template/provider benchmark runs are no longer
  `token_claim_eligible`, even when provider-reported token usage exists.
- Headless benchmark prompts no longer depend on clipboard or TUI multiline
  paste behavior for baseline failure context.
- Handoff verification summaries now prefer concrete planned verification
  commands, for example `npm test passed` or `npm test failed`, instead of
  using the final line of a multiline benchmark prompt as the receipt label.

## Benchmark Claim Rule

Use benchmark runs for public token-efficiency claims only when the artifact set
is complete and the exported run has:

```json
{
  "final_status": "verified_success",
  "execution_mode": "provider",
  "token_usage_source": "provider_reported",
  "token_claim_eligible": true
}
```

Estimated, local-template, mixed, failed, and unverified runs remain useful
engineering evidence, but they are not headline provider-token comparisons.

## Validation

Run from a clean public CLI clone:

```text
cargo fmt --all -- --check
cargo test --locked --workspace
cargo clippy --locked --workspace --all-targets -- -D warnings
cargo build --release --locked -p phonton-cli
target/release/phonton.exe version
npm run test:npm-wrapper
npm pack --dry-run
```
