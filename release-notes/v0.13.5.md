# Phonton CLI v0.13.5

v0.13.5 fixes the remaining generated chess benchmark failure where the first
existing-Vite rules slice could still ask a weak provider to invent a full
rules engine and return invalid TypeScript.

## Fixed

- Existing Vite/React chess runs now use a local verified rules seed for
  `src/chessRules.ts` and `src/chessRules.test.ts`.
- Partial failed reruns with broken `src/chessRules.ts` or
  `src/chessRules.test.ts` are repaired by replacing those artifacts with the
  local seed before provider dispatch.
- The first seed slice reports zero provider input/output tokens; provider
  calls start with UI integration slices.
- Existing-workspace chess token estimates exclude the local seed slice.
- Release binary builds no longer restore cached cargo binaries, avoiding
  stale cargo/rustup shims on macOS runners.
