# Phonton CLI v0.7.0

Trust loop receipts release.

## Added

- `phonton plan` text output now renders the visible GoalContract before work starts: acceptance criteria, expected artifacts, likely files, verification plan, run plan, quality floor, assumptions, and clarifications.
- `phonton demo trust-loop --json` provides deterministic onboarding evidence for the contract -> verification -> receipt -> memory loop without requiring provider credentials.
- `phonton review --markdown` exports a review receipt suitable for PR descriptions, benchmark artifacts, and release notes.
- `phonton run [latest|<task-id>]` runs receipt-suggested structured commands through the existing sandbox and reports exit code, duration, stdout, and stderr previews.

## Changed

- `phonton plan` now shares the same stack-aware contract preflight used by the TUI, so npm, Cargo, and Makefile workspaces show matching verification and run plans.
- Documentation now centers the trust-loop workflow and avoids benchmark superiority claims without reproducible artifacts.
