# Phonton CLI v0.16.0 Context, Extensions, And Website Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v0.16.0 as a proof-carrying ADE release with typed `@...` context mentions, clearer token accounting, stronger extension/MCP diagnostics, and a redesigned website that matches the CLI trust loop.

**Architecture:** Keep the CLI as the release anchor. Add typed mention records in `phonton-types`, resolve local context in `phonton-cli`, reuse existing `phonton-context` token counters and `phonton-mcp` trust metadata, then feed mention output into preflight, context, token, and handoff surfaces. Redesign the website in the separate website repo without mixing website files into the public CLI release clone.

**Tech Stack:** Rust, Tokio, Ratatui, serde, `phonton-types`, `phonton-context`, `phonton-mcp`, `phonton-extensions`, Docusaurus, React, CSS modules/global CSS, Playwright browser verification.
