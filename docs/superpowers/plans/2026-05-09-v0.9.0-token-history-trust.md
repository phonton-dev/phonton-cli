# v0.9.0 Token History Trust Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build v0.9.0 as a balanced release that lowers prompt-token waste and improves history/trust management.

**Architecture:** Add typed prompt-budget fields on top of the existing `PromptContextManifest`, deduplicate code context before rendering worker prompts, and make workspace trust/history visible through existing TUI command paths. Keep the implementation incremental; do not rewrite the orchestrator.

**Tech Stack:** Rust workspace, Ratatui TUI, rusqlite store, serde-serialized types, existing GitHub/npm release workflow.

---

## File Structure

- Modify `phonton-types/src/lib.rs`: extend `PromptContextManifest`; add `WorkspaceTrustRecord` and `WorkspaceTrustSource`.
- Modify `phonton-worker/src/lib.rs`: lean worker prompt, context dedupe, compact repo context rendering, richer manifest estimates.
- Modify `phonton-worker/src/dispatcher.rs`: avoid duplicate semantic retrieval where possible and report deduped context.
- Modify `phonton-cli/src/trust.rs`: load, record, list, revoke, and migrate richer workspace trust records while preserving legacy JSON.
- Modify `phonton-store/src/lib.rs`: optional workspace trust persistence and history helper queries.
- Modify `phonton-cli/src/tui_commands.rs`: parse `/trust` and richer `/history` commands.
- Modify `phonton-cli/src/main.rs`: TUI summaries, intent handling, history detail rendering, trust status display.
- Modify `README.md`, `CHANGELOG.md`, and `release-notes/v0.9.0.md`: public-facing release notes.
- Modify `package.json`, `phonton-cli/Cargo.toml`, `Cargo.lock`: version bump to `0.9.0`.
